# 远程推理系统 - 电脑运行YOLO，树莓派显示

## 系统架构

```
树莓派 (客户端)         电脑 (服务器)
     |                       |
摄像头捕获图像  ----->   YOLO模型推理
     |                       |
显示检测结果  <-----   返回检测结果
```

## 文件说明

### 1. 服务器端 (电脑运行)
- `remote_inference_server.py` - 主服务器，运行YOLO模型
- 需要：Python 3.7+, ultralytics, opencv-python

### 2. 客户端端 (树莓派运行)
- `raspberry_client.py` - 完整功能客户端
- `raspberry_client_ultra_light.py` - 极致精简版，适合树莓派4B
- 需要：Python 3.7+, opencv-python

## 极简使用指南

### 方法1：一键启动（推荐）
```bash
# 在电脑或树莓派上运行
start_remote_inference.bat
```

### 方法2：分步操作

#### 步骤1: 电脑端启动服务器
```bash
python remote_inference_server.py
```
**记下显示的IP地址**（如：192.168.1.100）

#### 步骤2: 树莓派端启动客户端
```bash
# 完整版（功能全面）
python raspberry_client.py

# 极致精简版（推荐树莓派4B）
python raspberry_client_ultra_light.py
```
输入电脑的IP地址即可连接。

## 配置说明

### 服务器配置 (remote_inference_server.py)
```python
server = RemoteInferenceServer(
    host='0.0.0.0',      # 监听所有网络接口
    port=9999,           # 监听端口
    model_path='apple_sensitive.pt'  # 模型文件
)
```

### 客户端配置 (raspberry_client.py)
```python
client = RaspberryClient(
    server_ip='192.168.1.100',  # 电脑服务器IP
    server_port=9999,           # 服务器端口
    camera_id=0                 # 树莓派摄像头ID
)
```

## 网络要求

1. **同一局域网**：电脑和树莓派必须在同一网络
2. **IP地址**：确保电脑防火墙允许9999端口
3. **网络延迟**：建议<50ms延迟以获得流畅体验

## 性能优化

### 树莓派端：
- 使用极致精简版客户端
- 降低分辨率：320x240
- 降低帧率：10-15 FPS
- 启用跳帧：每2帧处理1次

### 电脑端：
- 确保有足够GPU内存
- 使用轻量级模型（如yolov8n）
- 调整推理批处理大小

## 故障排除

### 1. 连接失败
- 检查电脑和树莓派IP地址
- 确认电脑防火墙允许9999端口
- 使用 `ping` 命令测试网络连通性

### 2. 延迟过高
- 降低图像分辨率
- 增加跳帧数
- 检查网络带宽

### 3. 检测不准确
- 确保模型训练充分
- 调整置信度阈值（默认0.1）
- 检查摄像头对焦和光照

## 操作说明

### 树莓派客户端：
- 按 `q` 键退出程序
- 显示信息：FPS、推理时间、检测数量
- 连接状态：绿色（已连接）/红色（断开）

### 电脑服务器：
- 按 `Ctrl+C` 停止服务器
- 显示统计：处理请求数、平均推理时间

## 扩展功能

### 1. 多客户端支持
服务器支持多个树莓派同时连接，每个客户端独立处理。

### 2. 模型热切换
可以在不重启服务器的情况下切换不同模型。

### 3. 性能监控
实时显示每个客户端的请求频率和延迟。

## 注意事项

1. **首次运行**：确保电脑和树莓派安装了必要的Python包
2. **模型文件**：确保模型文件存在且路径正确
3. **摄像头权限**：树莓派需要摄像头访问权限
4. **网络稳定性**：不稳定的网络会导致连接中断
